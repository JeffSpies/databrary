It extends ActionRoute, which already has most info about a route and how to serve it, to include the authentication requirement.

The actual web app

The second level of the Databrary 'web framework'. Makes a (wrapped) Wai Application given a route map, a hatch into the Wai Route fallback, and the already-generated system capabilities.

Most routes are served by web-inv-route—style ActionRoutes, but some (~30%) have been converted to Wai Routes.
